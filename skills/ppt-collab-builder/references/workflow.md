# 6阶段工作流详细说明

## 阶段1: 需求分析

**目标**：收集基础需求，AI生成PPT结构方案供用户确认

**需要询问的问题**：
1. 项目名称（例如：差旅用品经营分析）
2. PPT的主要目的（向客户展示、内部汇报、竞品分析等）
3. 目标受众是谁（客户、管理层、销售团队等）
4. 您认为这份PPT中**必须包含**的板块有哪些？（例如：生意概览、品类分析、目标达成、增长机会等）

**AI生成方案**：
根据用户提供的必需板块，AI生成2-3个完整的PPT结构方案，每个方案包含：
- 建议的页面顺序
- 每页的标题和简要说明
- 方案的优缺点分析

用户选择一个方案或提出修改意见，最终确认PPT结构。

**输出**：
- 项目基本信息
- 确认后的页面大纲（每页标题+作用）

---

## 阶段2: 大纲完善

**目标**：确定项目风格画像、视觉规范、每页详细配置和版式策略

**需要确认的内容**：
1. 选择基础主题风格（商务蓝/business、简约灰/simple、科技绿/tech）
   - **提示**：您是否有参考的风格截图可以上传？我们可以基于截图来调整视觉风格
   - 基础主题只是起点，不是最终固定模板
2. 生成项目风格画像（AI先建议，再与用户确认）：
   - 使用场景（内部汇报、客户提案、经营复盘、产品展示等）
   - 目标受众（管理层、客户、销售团队、运营团队等）
   - 视觉气质（克制专业、提案叙事、科技现代、品牌陈列等）
   - 信息密度（高/中/低）
   - 版式节奏（数据密集页、结论页、章节页、图片页如何交替）
   - 适合使用和应该避免的版式
3. 确认视觉规范（尺寸、边距、字体、色彩、背景、图表样式等）
4. 逐页细化（AI先给出建议，再与用户确认）：
   - 页面类型（overview/efficiency/structure/essential/super_3d/super_performance/targets/growth/investment）
   - 页面任务（结论强调、趋势分析、结构拆解、对比判断、商品展示、机会识别等）
   - 版式变体（从`references/adaptive-layout.md`和`references/templates.md`中选择或组合）
   - 页面目的（1-2句话描述）
   - 页面布局（左右分栏、上下布局、全屏等，详细描述各区域占比）
   - 是否需要图表
   - 是否需要表格
   - 是否需要图片
   - 内容排版比例

**最终大纲必须包含**：
- **项目风格画像与版式策略**（表格，包含：使用场景、目标受众、视觉气质、信息密度、版式节奏、适合/避免的版式）
- **整体视觉规范**（详细表格，包含：设计风格、页面尺寸、边距、标题字体、正文字体、主色、辅助色、警告色、页面背景、卡片背景等）
- **每页结论呈现规范**（表格，包含：位置、样式、用途，以及不同版式下结论如何呈现）
- **每页详情**：
  - 页面目的
  - 页面任务
  - 版式变体
  - 页面布局（详细描述，包含各区域占比、图表类型、数据系列等）
  - 涉及的图表编号（如：CHART_01_001, CHART_01_002）
  - 洞察内容占位（可选，用于后续数据洞察填充）

**输出**：
- 详细内容大纲（含每页完整配置）
- 项目风格画像与版式策略
- 视觉规范定义
- 结论呈现规范

---

## 阶段3: 图表规划

**目标**：识别需要数据展示的页面，AI先判断呈现方式（纯表格vs ECharts），再给出建议方案确认

**需要确认的内容**：
对每个需要数据展示的页面，按以下顺序处理：

### 第一步：呈现方式决策（AI先建议）
AI首先判断并说明推荐使用哪种呈现方式：

**纯表格（直接用pptxgenjs生成）**：
- 适用场景：数据清单、详细指标对比、SKU列表、多维度小数据量表格
- 数据特征：行数≤15行，列数≤8列，以展示精确数值为主
- 典型页面类型：super_3d（超单3D选品）、essential（必备品全景数据部分）、structure右侧数据表格

**ECharts复杂图表（需预生成图片）**：
- 适用场景：趋势分析、占比分析、对比分析、分布分析、象限分析
- 数据特征：需要可视化趋势、对比、分布关系，数据量较大
- 典型页面类型：overview（指标卡片除外）、efficiency、structure左侧饼图、super_performance、growth、investment、targets

### 第二步：根据呈现方式确认细节

**如果是纯表格**：
1. 表格列定义（列名、数据类型、宽度比例）
2. 样式规范（表头样式、数据对齐方式、是否需要斑马纹）
3. 标记为`TABLE_`开头的编号（如：TABLE_01_001）

**如果是ECharts复杂图表**：
1. 图表数量
2. 每个图表的（AI先建议）：
   - 图表类型（折线、柱状、饼图、面积、散点等）
   - X轴维度（例如：月份、渠道、品类）
   - Y轴维度（例如：销售额、店铺数、增长率）
   - 数据系列（用逗号分隔，例如：销售额、订单量、客流）
   - 数据来源说明（可选）
3. 标记为`CHART_`开头的编号（如：CHART_01_001）

对需要图片的页面：
1. 需要哪些图片的描述

**输出**：
- 数据展示规划清单（含呈现方式、确认后的维度定义）
- 图片需求清单
- 明确区分TABLE_和CHART_编号的资源清单

---

## 阶段4: 资源准备

**目标**：生成数据模板Excel文件、创建图片文件夹结构

**执行内容**：
1. 创建项目目录结构
2. 为每个图表生成Excel模板（含说明页+数据页）
3. 为每个需要图片的页面创建对应文件夹
4. 告知用户如何填充数据和放置图片

**输出**：
- Excel数据模板文件
- 图片文件夹结构
- 资源准备指南

---

## 阶段5: 数据洞察

**目标**：读取已填充的数据，生成关键结论，确认数据质量

**执行内容**：
1. 读取用户填好的Excel文件
2. 自动分析数据：
   - 峰值/谷值识别
   - 趋势判断（上升/下降/平稳）
   - 增长率计算
   - 平均值/最大值/最小值
3. 生成数据洞察文案
4. 与用户确认洞察是否准确，可补充业务洞察
5. 更新完善内容大纲（将洞察填入对应页面）

**输出**：
- 数据分析结果
- 数据洞察文案
- 更新后的内容大纲

---

## 阶段6前置：环境检查

**目标**：确保Playwright和浏览器内核已正确安装，否则无法生成ECharts图表

**检查清单**：

1. **检查Playwright是否安装**
   ```bash
   npm list playwright
   # 或
   yarn list playwright
   ```

2. **检查浏览器内核（Mac用户优先检查Chrome）**
   ```bash
   # 检查本地Chrome
   ls "/Applications/Google Chrome.app"
   ```

3. **如未安装，按以下步骤操作**：

   **Mac用户**：
   - 有Chrome → 直接使用，无需下载
   - 无Chrome → 运行 `npx playwright install chromium`

   **Windows/Linux用户**：
   - 运行 `npx playwright install chromium`

4. **验证安装**
   ```bash
   node -e "const playwright = require('playwright'); console.log('✅ Playwright环境OK');"
   ```

**重要提醒**：环境检查不通过，不要进入阶段6！

---

## 阶段6: 最终生成

**目标**：按顺序执行数据洞察→检查风格与版式适配→生成ECharts（仅复杂图表）→合成最终PPT

**技术栈要求（严格执行）**：
- **必须使用**：JS + ECharts(浏览器渲染) + pptxgenjs
- **禁止使用**：Python或其他技术栈方案
- **重要说明**：ECharts复杂图表不使用pptxgenjs内置图表功能，使用浏览器(Puppeteer/Playwright) + ECharts库渲染生成高清PNG图片

**执行步骤（必须按顺序）**：

**步骤1：数据洞察与大纲更新**
- 读取已填好的数据
- 执行数据分析，生成洞察文案
- 将洞察更新到内容大纲的对应页面
- 确认大纲完整无误

**步骤2：视觉风格与版式适配检查**
- 再次检查项目风格画像是否已经明确
- 再次检查视觉规范是否已经明确
- 检查每页是否已经选择版式变体和内容比例
- 如果未明确或用户未确认，再次与用户确认
- 对全稿做版式单一性检查：
  - 是否连续多页使用相同标题栏、白卡片、左右分栏
  - 是否所有洞察都放在同一位置导致节奏单调
  - 数据密集页、结论页、图片页、对比页是否有不同视觉节奏
  - 图表、表格、图片、洞察的占比是否匹配页面目的
- 根据检查结果调整页面布局，但不改变用户已确认的数据逻辑和图表定义
- 确认后锁定视觉规范、版式策略和每页布局

**步骤3：预生成ECharts复杂图表（仅复杂图表）**
- **仅处理**标记为`CHART_`开头的ECharts复杂图表
- **跳过**标记为`TABLE_`开头的纯表格内容
- **重要**：不使用pptxgenjs内置图表功能
- 使用**浏览器(Puppeteer/Playwright) + ECharts库**渲染图表
- 为每个ECharts图表生成配置
- 确保图表符合大纲中的排版比例要求
- 检查图表内文字不要超出图表范围（自动调整字体大小、标签角度等）
- 生成2倍分辨率(@2x)高清PNG图片
- 保存到临时目录供预览

**步骤4：合成最终PPT**
- 使用pptxgenjs生成PPT
- 按项目风格画像和每页版式变体应用布局
- **纯表格处理**：直接使用pptxgenjs表格API生成，无需插入图片
- **ECharts图表处理**：插入预生成的高清图片
- 插入图片、洞察文案
- 应用视觉规范（主题色、字体等）和版式策略（标题形态、内容比例、洞察位置、页面节奏）
- 保存最终PPT文件

**输出**：
- 高清ECharts图表图片（仅复杂图表）
- 最终PPT文件
- 项目摘要JSON

---

## 状态流转

```
阶段1 (需求分析) → AI生成结构方案 → 用户确认
    ↓
阶段2 (大纲完善) → 可上传风格截图 → 生成项目风格画像 → AI建议每页版式配置 → 确认最终大纲(含风格画像/视觉规范/结论规范/每页详情)
    ↓
阶段3 (图表规划) → AI先判断:纯表格还是ECharts? → AI建议方案 → 用户确认
    ↓
阶段4 (资源准备) ← 用户填写数据、放置图片
    ↓
阶段5 (数据洞察) → 更新大纲
    ↓
阶段6 (最终生成) → 步骤1:数据洞察更新大纲 → 步骤2:检查风格与版式适配 → 步骤3:浏览器+ECharts渲染(表格跳过) → 步骤4:用JS+ECharts(浏览器)+pptxgenjs生成最终PPT
    ↓
完成！
```

每个阶段都支持返回修改，用户可以在确认时选择"否"来重新进行该阶段。
