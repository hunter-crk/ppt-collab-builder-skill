# 自适应排版规则

本文件用于让agent在不改变6阶段工作流和技术栈的前提下，根据不同项目风格生成更匹配的PPT版式。

## 核心原则

- 项目风格画像决定视觉系统，页面任务决定单页版式。
- 基础主题（business/simple/tech）只是起点，不是固定模板。
- 不要让整份PPT重复同一种标题栏、白卡片、左右分栏。
- 保持统一的字体、颜色、图表语言和数字样式；允许标题形态、模块比例、洞察位置随页面任务变化。
- 版式变化必须服务信息表达，不能为了变化而牺牲阅读顺序。

## 阶段2必须产出：项目风格画像

在大纲完善阶段，先给出项目风格画像并让用户确认。建议使用以下表格：

| 项目 | 策略 |
|------|------|
| 使用场景 | 内部汇报/客户提案/经营复盘/产品展示/项目总结等 |
| 目标受众 | 管理层/客户/销售团队/运营团队/投资人等 |
| 视觉气质 | 克制专业/提案叙事/科技现代/品牌陈列/高密度运营看板等 |
| 信息密度 | 高/中/低，并说明原因 |
| 版式节奏 | 数据密集页、结论页、图片页、章节页如何交替 |
| 标题策略 | 顶部栏/无栏大标题/左侧竖栏/章节标题等 |
| 洞察策略 | 标题下结论/右侧洞察/底部总结/图上标注/行动建议区等 |
| 适合的版式 | 从通用版式变体中选择 |
| 避免的版式 | 说明哪些版式不适合本项目 |

## 项目类型到风格策略

| 项目类型 | 推荐风格 | 信息密度 | 常用版式 | 避免 |
|----------|----------|----------|----------|------|
| 管理层经营分析 | 克制、清晰、数据优先 | 高 | `dashboard_grid`, `chart_focus`, `split_insight`, `dense_table` | 大面积装饰、过多图片、低信息密度 |
| 客户提案/销售汇报 | 叙事清楚、结论突出、留白适中 | 中 | `hero_kpi`, `split_insight`, `before_after`, `narrative_steps` | 满屏小字、连续表格页 |
| 运营复盘 | 诊断感强、问题和动作并重 | 高 | `dashboard_grid`, `split_insight`, `dense_table`, `matrix_quadrant` | 只展示现象不展示动作 |
| 品牌/商品展示 | 图片权重高、指标辅助 | 中/低 | `image_catalog`, `hero_kpi`, `before_after`, `split_insight` | 纯表格堆砌、图片过小 |
| 科技/产品方案 | 模块化、现代、层级清晰 | 中 | `narrative_steps`, `split_insight`, `matrix_quadrant`, `chart_focus` | 传统重标题栏、装饰复杂 |
| 投资/ROI分析 | 逻辑严谨、数字突出 | 高 | `chart_focus`, `before_after`, `dense_table`, `split_insight` | 过度营销化、结论不落到数字 |

## 页面任务到版式变体

| 页面任务 | 首选版式 | 备选版式 | 结论呈现建议 |
|----------|----------|----------|--------------|
| 开场总览 | `hero_kpi` | `dashboard_grid` | 主标题下放一句核心结论，数字做主视觉 |
| 多指标概览 | `dashboard_grid` | `split_insight` | 每个指标给短标签，整体洞察放右侧或底部 |
| 趋势分析 | `chart_focus` | `split_insight` | 图表上或右侧标注峰值、拐点、趋势判断 |
| 结构占比 | `chart_focus` | `split_insight`, `before_after` | 突出最大项、变化项和业务含义 |
| 明细清单 | `dense_table` | `split_insight` | 表格高亮关键行，旁边或底部给筛选结论 |
| 商品展示 | `image_catalog` | `split_insight` | 图片优先，指标作为标签，不要压缩图片到不可辨认 |
| 竞品/方案对比 | `before_after` | `split_insight` | 左右或上下严格对齐，突出差异和选择建议 |
| 机会判断 | `matrix_quadrant` | `split_insight` | 象限内标重点对象，右侧列行动优先级 |
| 策略路径 | `narrative_steps` | `section_divider` | 每一步只保留一个动作和一个判断依据 |
| 章节过渡 | `section_divider` | `hero_kpi` | 用短标题和关键句控制节奏 |

## 版式变体定义

### `hero_kpi`

- 结构：大结论 + 1-3个核心数字 + 少量解释。
- 适合：首页、阶段总结、关键结论页。
- 注意：不要放超过5个指标；否则改用`dashboard_grid`。

### `dashboard_grid`

- 结构：2x2、3x2或主图+小图组合。
- 适合：经营概览、运营复盘、多指标监控。
- 注意：统一数字格式，图表标题要短，避免每个模块都写长段落。

### `chart_focus`

- 结构：单一大图表占60%-80%页面，结论在标题下、图上标注或底部。
- 适合：趋势、结构、分布、ROI等需要一眼读懂的图。
- 注意：减少装饰和多余边框，保证坐标轴和标签可读。

### `split_insight`

- 结构：主图/表占60%-70%，洞察/动作建议占30%-40%。
- 适合：分析页、诊断页、客户汇报页。
- 注意：右侧洞察不要只是复述图表，要给原因、风险或动作。

### `before_after`

- 结构：左右或上下对照，指标、标题和图表位置严格对齐。
- 适合：竞品对比、改善前后、方案A/B。
- 注意：对比维度要一致，不要让两侧内容量差异过大。

### `matrix_quadrant`

- 结构：象限/矩阵为主，配重点标注和机会列表。
- 适合：增长机会、选品优先级、客户分层。
- 注意：象限标签要业务化，避免只写“高/低”。

### `image_catalog`

- 结构：图片网格、图片+指标标签、图片+简短推荐理由。
- 适合：商品、品牌、案例、SKU展示。
- 注意：优先保证图片清晰，指标做辅助，不要把图片页做成表格页。

### `narrative_steps`

- 结构：横向路径、纵向流程、阶段推进。
- 适合：策略、计划、方法论、执行路径。
- 注意：每步只表达一个动作，数字依据可作为脚注或小标签。

### `dense_table`

- 结构：全幅表格或表格+重点结论。
- 适合：SKU清单、多维指标、精确数值比较。
- 注意：列数超过8或行数超过15时，应分页、分组或改为摘要+附录。

### `section_divider`

- 结构：章节名、关键句、少量背景元素。
- 适合：长PPT章节过渡。
- 注意：不要占用过多页数；用于建立阅读节奏。

## 阶段6版式审查清单

最终生成前必须检查：

1. 项目风格画像是否存在并已确认。
2. 每页是否写明页面任务和版式变体。
3. 连续3页是否使用了完全相同的标题形态和内容布局。
4. 是否存在所有页面都使用白色卡片包裹内容的问题。
5. 结论位置是否有节奏变化：标题下、侧栏、底部、图上标注、行动建议区。
6. 图表、表格、图片的面积是否匹配页面任务。
7. 数据密集页的小字是否可读，表格是否过密。
8. 图片页是否保证图片清晰可辨。
9. ECharts图表是否按版式比例生成，而不是生成后强行拉伸。
10. 全稿颜色、字体、数字格式、图表轴线和表格样式是否统一。

如果检查发现版式过于单一，优先调整页面版式和洞察位置，不改变已经确认的数据口径、图表定义和业务结论。
